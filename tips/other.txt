Tips for solving a problem

Ideas:
    1. Try to make observations and simplify the problem
    2. Can it be modeled as a graph?
    3. Can it be solved using DP?
        3.1. Bitmask DP
        3.2. Interval DP
        3.3. Tree DP
        3.4. DP with prefix/suffix optimization
    4. Can it be solved with binary search?
    5. Can it be solved using a standard algorithm?
    6. Can you use greedy?
    7. Back to 1.

    *if the idea is too hard to implement (like a dp with a lot of edge cases), maybe you are on the wrong path

WA:
    1. Verify the logic, if you're not sure about it
    2. Verify all the small mistakes (in the entire code, it may have more than one typo)
        2.1. Missing return statements or endl on special cases
        2.2. i instead of j
    3. Verify small cases
    4. TYPO (in index, specially)

Contest tips:
    1. Don't try to "hope code" a problem, this is, build a frankstein code that you spent like 5 seconds trying to prove it was right and 30min writing it, and then you submit with the hope that it works and spend the rest of the contest trying to fix it, while getting many WAs.
    2. try to think the most possible before writing the code. this avoid a lot of headaches after. even writing a pseudocode is good practice
